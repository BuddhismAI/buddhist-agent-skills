# Question Patterns

Recurring question shapes and their decomposition recipes. Consulted **after** the Question Analysis Scaffold in `SKILL.md` §6 has run steps ①–④ (Surface / Register / Context / Underlying need). A match supplies aspect + routing hints and `Don'ts`; absence doesn't block answering.

Patterns are indexed by surface signals. Each entry follows a fixed template:

```
### Pattern: <short name>
**Signals:** surface cues that identify this shape
**Likely confusion / need:** the real gap or real need driving the question
**Aspects to cover:** what the answer should address
**Topic routing:** topic → specific subtopic docs (use topic-index concept routers)
**Don'ts:** common wrong approaches
```

Two clusters: **doctrinal** (interpretive / philosophical recurring confusions) and **practical** (life-oriented questions asked by real practitioners). Patterns grow based on observed failures — when a new recurring shape appears, add a seed.

Cross-references to topic index docs live in the routing line; these patterns do not duplicate the topic's own correctness anchors.

## Doctrinal cluster

### Pattern 1: 空性 query whose real confusion is 单空 vs 离戏空

**Signals:** 用户将 空性 / 无自性 / 四大皆空 理解为 "什么都没有" / "瓶子不存在"；或把 无自性 当作胜义终点；或以 "我已经知道空性是空" 的姿态追问下一步。

**Likely confusion:** 把 单空（相似胜义）当作真实胜义；未区分 "分别念所缘的无" 与 "离戏大空性"。

**Aspects to cover:**
- 单空（相似胜义）与离戏大空性（真实胜义）的区别；
- 单空作为过渡阶段的正当作用 —— 初学者可先止于此，但不可止步于此；
- 四边戏论穷尽才是真实胜义；
- 从单空过渡到离戏的实修进路（观察 → 定解 → 四边遣除 → 安住）。

**Topic routing:** madhyamaka → `debates.md` §单空vs离戏空; `classifications.md` §二谛; drill: `collections/中论/品15-观有无品.md`, `collections/中论/品18-观我法品.md`, `collections/中论/品24-观四谛品.md`, `collections/中观庄严论/20-遣除胜义争论.md`。

**Don'ts:** 不以 "空性就是什么都没有" 开头；不以龙树传记铺垫；不只列颂词而不点出 相似/真实 二分；不默许用户把单空当作终点。

### Pattern 2: 应成 vs 自续 reduced to one axis

**Signals:** 用户问 "应成和自续有什么区别?" 但仅提到 有承认/无承认 一轴；或暗含 "谁更高" 的格鲁派式排序；或把 "无承认" 理解为应成派完全不说话。

**Likely confusion:** 把多轴分判压缩成一维差别；忽略宁玛派对二者的互补定位。

**Aspects to cover:**
- 所破范围（应成破四边与承许，自续主破谛实分）；
- 能破方式（应成四大应成因、自续三相推理）；
- 二谛安立方式（名言中是否建立自相分位）；
- 入定出定的不同侧重（应成重入定无执，自续重出定建立）；
- 宁玛派对二者的整合视角（非高下，而是不同侧面）。

**Topic routing:** madhyamaka → `debates.md` §应成自续; `classifications.md` §五种分类; drill: `collections/中论/品01-观因缘品.md` §分流点, `collections/中观庄严论/09-应成自续与总义收结.md`。

**Don'ts:** 不把自续派定位为较低宗派；不单讲 "有无承认" 而不提多轴；不回避 宁玛派整合视角；不借格鲁派框架作答。

### Pattern 3: 如何修X — narrow-mechanical vs essential definition

**Signals:** 用户问 "如何修 [布施 / 安忍 / 持戒 / 精进 / 禅定]"，并在描述中把该度理解为狭义机械动作 —— 发钱=布施、不生气=安忍、念多少遍=持咒圆满、远离人群=修安忍。

**Likely confusion:** 以狭义形式定义代替心要；错过 "心 胜于 形" 的核心。

**Aspects to cover:**
- 该度/法的心要（e.g., 布施 = 舍心；安忍 = 护菩提心不退；持戒 = 防非止恶之意乐）；
- 为何仅形式不足以转化烦恼（可引 "牛饮水成乳，蛇饮水成毒"）；
- 形式与心要如何配合，形式在何种条件下成为圆满；
- 具体例子 —— 听法中也可圆满布施（陪听者供养道友）；闭关独修若不接触对境反而不能真修安忍。

**Topic routing:** foundations → `practice-path.md`, `bodhicitta-methods.md`; drill: `collections/入行论/` 对应品（布施→品01, 持戒→品05, 安忍→品06, 精进→品07, 静虑→品08, 智慧→品09）; `collections/修心七要/要04-归结终生修法.md`（五力）, `collections/佛子行/10-修学六度.md`（学处·六度 颂25–30）。

**Don'ts:** 不以配方式清单作答；不跳过 心/形 的对照；不把 "多做X遍" 当作圆满标准。

### Pattern 4: Conceptual question about experiential topic (stage mismatch)

**Signals:** 用户以概念方式追问 定解 / 安住 / 证悟 / 智慧（"什么是定解?"、"开悟是什么感觉?"）；或以 "我已经证悟了" 开头；或追问 "畏因果与证悟空性是否矛盾"。

**Likely confusion:** 把 闻思阶段的概念理解 误当作 修证阶段的体验；或把 修证 看成遥不可及；或忽略 真正证悟者取舍因果更细致 的反向指标。

**Aspects to cover:**
- 词句能触及什么，不能触及什么（"说似一物即不中"）；
- 观察 → 定解 → 安住 → 交替 → 任运 的次第；
- 用户当前应立足的入口阶段（若用户在闻思阶段而问证悟，应指向 观察 而非 安住）；
- 畏因果与证悟空性并不矛盾 —— "见解越高，取舍因果越细致"；自称证悟而不畏因果恰是未证悟的征相。

**Topic routing:** madhyamaka → `reasoning-methods.md` §运用能破抉择心的本性 / §从观察到定解; foundations → `practice-path.md` / `key-quotes.md`。

**Don'ts:** 不假装概念语言可以替代体验；不跳过用户应先立足的阶段；不验证用户声称的境界；不回避 畏因果 这一基础指标。

### Pattern 5: Cross-topic question treated as single-topic

**Signals:** 问题跨越两个主题（因明 ∩ 中观、如来藏 ∩ 中观、俱舍 ∩ 中观）但用户以单主题术语追问；或在一个主题框架下滑入另一主题概念。

**Likely confusion:** 不同主题有不同所破所证与术语系统，混合会失去精度；宁玛派看待二者常为 圆融 而非对立，但不代表 一致。

**Aspects to cover:**
- 各主题在此问题中分别在讨论什么（分开说清）；
- 交集点在哪里（术语对应、论证对应）；
- 必要时分别给出两方回答；
- 宁玛派如何看待二主题的关系 —— 通常是 圆融（如因明为中观服务；如来藏与离戏大空性不二）而非对立。

**Topic routing:** 相关的所有 topic `index.md`; `references/maps/topic-index.md` for topic overview; drill into cross-topic material in `collections/中观庄严论/`（二理妙车 combines yogachara + madhyamaka）、`collections/解义慧剑/`（pramana + madhyamaka）。

**Don'ts:** 不把两主题并合为单一回答；不只用一方术语去解读另一方；不把 圆融 当作 一致。

## Practical / life cluster

### Pattern 6: Life-situational

**Signals:** 用户描述自身或亲人的现实处境（亲人离世 / 生重病 / 焦虑 / 失业 / 感情困扰 / 人生迷茫），并夹带 "佛教怎么看?" / "我该怎么办?"。

**Likely need:** 温度 + 佛法框架 + 可落地的下一步 —— 不是胜义讲义。

**Aspects to cover:**
- 先承认对方处境（情感连接，不急于教学）；
- 一个可立住的佛法视角（如 无常、缘起、业因），不展开论证；
- 面向当下的一个具体修持建议（如 自他相换、金刚萨埵忏悔、念佛、古萨里断法）；
- 鼓励并提示 —— 若需深入，依止合格善知识。

**Topic routing:** foundations → `collections/入行论/品06-*.md`（面对痛苦）/ `collections/修心七要/要03-恶缘转道用.md` / `collections/佛子行/06-恶缘转道用·四不欲.md`, `07-恶缘转道用·两难忍与兴衰.md`, `08-恶缘转道用·贪嗔.md`; `collections/大圆满前行/寿命无常·*.md`, `轮回过患·*.md`, `因果不虚·*.md`, `金刚萨埵.md`。

**Don'ts:** 不以 "一切皆空" / "胜义中无此事" 开头；不把用户处境当作教学例子；不扮演修行大师角色；不越过用户的情感层直接进入教理。

### Pattern 7: Stage / progress self-assessment

**Signals:** "我学了X年怎么还是..."、"我现在是什么境界?"、"是不是已经开悟了?"；或期待明确的进度指标；或声称某种体验要求确认。

**Likely confusion/need:** 把外在标记（课程数 / 时间）当作修行进度；期待特定体验作为证明；或担心自己毫无进步；需要的是 诊断 + 调整建议，不是验证。

**Aspects to cover:**
- 自我评估为何常常失真（上师如意宝教言：自称证悟者往往未证）；
- 真实进步的征相（烦恼自然减轻、取舍因果更细致、悲心自然生起、顺违不扰）；
- 假进步的警号（傲慢自负、空口否定因果、谈神通、排斥闻思）；
- 用户当前位置的诊断问题（对什么境况仍生嗔？ 是否仍坚持取舍因果？）；
- 调整建议（增加什么观修，减少什么形式依赖）。

**Topic routing:** foundations → `practice-path.md` / `key-quotes.md`; madhyamaka → `reasoning-methods.md` §从观察到定解; drill: `collections/大圆满前行/闻法方式.md`（闻法动机相关）, source corpus for 修行得果征相 specifics。

**Don'ts:** 不直接验证或否认用户声称的境界；不用单一指标（时长、课数）评判；不忽略 畏因果 这一基础指标；不替用户预言"还要X年"。

### Pattern 8: Karma-cause inquiry

**Signals:** "我/他 为什么 [生病 / 贫穷 / 遭遇不幸 / 感情不顺]?"、"这是不是前世..."、"做了X会不会下地狱?"、"XX事件有什么果报?"。

**Likely confusion:** 三种误区之一 —— 宿命论（业=不可转）、道德审判（业=惩罚）、或完全否认（把一切归为偶然 / 外缘）。

**Aspects to cover:**
- 业是众缘之一而非唯一因 —— 缘起不仅仅是业，今生外缘、努力、发愿都参与；
- 具体业因果对应（如有）仅作参考，不替代当下选择；
- 当下如何转道用（以逆缘为助缘，参修心七要 §恶缘转道用）；
- 破除宿命论 —— 忏悔 + 发愿 + 善行可转业（贫穷者通过上供下施感召福报的公案为证）；
- 不替他人诊断具体前世业因。

**Topic routing:** foundations → `collections/入行论/品06-*.md`（逆缘转道用）/ `collections/修心七要/要03-恶缘转道用.md` / `collections/佛子行/06-恶缘转道用·四不欲.md`, `07-恶缘转道用·两难忍与兴衰.md`, `08-恶缘转道用·贪嗔.md`; `collections/大圆满前行/因果不虚·*.md`。

**Don'ts:** 不指定具体前世业因；不把用户处境定性为 "应得"；不淡化业的现实作用；不以"一切是因果"替代具体分析。

### Pattern 9: Subtle-realm / personal experience

**Signals:** 打坐时出现光/声/异常感受；梦见佛菩萨或本尊；声称有感应 / 神通；遇到鬼神作障；他人自称神通；关于中阴 / 亡者的具体问题。

**Likely confusion:** 两极化 —— 要么轻信（追逐神通、执著境界），要么全盘否定（无视业力感召与护法加持），要么自我诊断为 X 境界。

**Aspects to cover:**
- 普遍原则：体验 ≠ 证悟；不执不弃；
- 正征 vs 魔事（菩提心增上、悲心自然、烦恼减轻为正征；骄慢、排斥、情绪反常、追求异相为魔事征相）；
- 对鬼神 / 作障应以悲心而非怨敌心看待（鬼魔比自己更苦）；
- 不自我诊断具体境界；
- 建议寻求合格上师确认，重大境界尤需如此。

**Topic routing:** foundations → `collections/大圆满前行/上师瑜伽.md`, `金刚萨埵.md`, `往生法.md`（中阴 context）, `日修闭关要诀.md`; `collections/入行论/品08-*.md`; source corpus for 鬼神公案 and specific 境界 diagnoses。

**Don'ts:** 不替对方判定 "这是X境界"；不验证或否认他人神通声称；不忽视用户现实体验；不鼓励追求异常体验；不在 subtle-realm 问题上强扮专家。

### Pattern 10: Beginner orientation / comparative framing

**Signals:** "我想开始学佛，从哪里入手?"、"藏传 / 汉传 / 南传 有什么区别?"、"先看什么书?"、"应该学哪个宗派?"；对多种选择感到无从下手。

**Likely confusion/need:** 需要一张 地图（主要传统 + 基本概念 + 切入点），而非一本专论；避免被繁杂的宗派 / 经典名阻退。

**Aspects to cover:**
- 共同基础（四圣谛、三法印、八正道、皈依、因果） —— 这是各传统共许的门槛；
- 主要传统简介及其各自侧重（汉传：禅/净/华严/天台；藏传：宁玛/噶举/萨迦/格鲁；南传：上座部）；不作高下判断；
- 一个具体的下一步（如：读《佛子行三十七颂》、听 大圆满前行 第一堂课）；
- 如何衡量与自己相应（法要能对治烦恼、生起出离与菩提心为相应征相）；
- 依止合格上师的重要性 —— 但先不急着归依任一门派。

**Topic routing:** `references/topics/foundations/index.md`（加行入门）; `references/maps/topic-index.md`（主题地图）; `references/maps/collection-index.md`（经论清单）; drill: `references/collections/佛子行/` 作为最短入门、`references/collections/大圆满前行/` 作为最系统入门。

**Don'ts:** 不堆砌完整宗派分类；不默认用户已懂藏传术语；不推销宁玛为唯一正道（即便这是本技能默认）；不假设用户已有汉传 vs 藏传 vs 南传 的先验偏好；不替用户选择宗派。
